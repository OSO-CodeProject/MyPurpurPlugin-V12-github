2. Race Condition в DeadlineScheduler 🔴
Проблема: Проверка и удаление deadlines не атомарны

java
public void cancelDeadline(@NotNull Team team) {
    boolean removed = deadlines.remove(teamId) != null;
    // Между этими операциями другой поток может изменить данные
    UUID leaderId = teamLeaderIds.remove(teamId);
}
Решение: Синхронизировать критические секции:

java
public void cancelDeadline(@NotNull Team team) {
    synchronized (deadlineLock) {
        boolean removed = deadlines.remove(teamId) != null;
        UUID leaderId = teamLeaderIds.remove(teamId);
        if (leaderId != null) {
            clearLeaderDisplay(leaderId);
        }
    }
    if (removed) {
        storage.markDeadlinesDirty();
    }
}

🟡 ВАЖНАЯ: Race condition при cleanup invites/requests
Проблема:

java
// MembershipService.java
private void clearInvitesForPlayer(@NotNull UUID playerId) {
    Map<UUID, PendingInvite> invites = invitesByPlayer.remove(playerId); // ❌
    if (invites == null) {
        return;
    }
    for (PendingInvite invite : invites.values()) {  // ❌ ConcurrentModificationException
        invitesByTeam.computeIfPresent(...);
    }
}
Решение:

java
private void clearInvitesForPlayer(@NotNull UUID playerId) {
    Map<UUID, PendingInvite> invites = invitesByPlayer.remove(playerId);
    if (invites == null) {
        return;
    }
    // Создать копию перед итерацией
    List<PendingInvite> invitesCopy = new ArrayList<>(invites.values());
    for (PendingInvite invite : invitesCopy) {
        invitesByTeam.computeIfPresent(...);
    }
}
Приоритет: 🟡 Средний
